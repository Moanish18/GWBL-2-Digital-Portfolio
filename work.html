Procurement RPA Bot ‚Äì Sesami to SharePoint Automation

An end-to-end Power Automate for Desktop flow that logs into the Sesami portal, extracts RFQ/Tender information, downloads response and analysis files, and uploads them into a structured SharePoint library.

Tech: Power Automate, SharePoint Role: RPA Developer Period: Apr ‚Äì Nov

High-level Automation Flow

The bot automates repetitive manual steps previously done by Procurement officers: logging into Sesami, retrieving RFQs/Tenders, downloading ZIP, PDF and Excel files, classifying them by department.

By taking over an in-progress flow from my supervisor and completing it, I helped to develop a production-ready solution that reduces human error and creates an auditable trail of procurement documents.

Brief Description of Deliverable

A single, robust Power Automate desktop flow that is triggered everyday automatically to process a batch of RFQs/Tenders from Sesami and sync them into a structured SharePoint repository without manual downloading or file-moving.

My Contribution

  • Refactored the incomplete original flow into clear segments (Sesami navigation, downloads, classification, upload).
  • Added data table cleaning, file-type classification rules, and resilient UI automation patterns.
  • Implemented exception handling so that single-file failures do not crash the entire run.

How It Will Be Used

The bot is intended to be run by the Procurement team to process RFQs/Tenders from Sesami on a regular basis, supporting future unattended deployment once licensing and VM setup are complete.
